1. 包、导入与模块

每个 .go 文件以 package 声明开头

package main       // 可执行程序的入口包
package model      // 普通库包，包名通常等于目录名

import 的三种形式

// 1. 普通导入
import "fmt"

// 2. 分组导入（项目中最常见）
import (
    "context"
    "fmt"

    "code.byted.org/gopkg/logs"                        // 内部依赖
    "github.com/cloudwego/eino/schema"                  // 第三方依赖
)

// 3. 别名导入
import (
    internal_common "code.byted.org/.../internal/common"  // 解决包名冲突
    dalRedis        "code.byted.org/.../internal/platform/dal/redis"
)
// 使用时: internal_common.AsyncSendLarkAlarm(...)

导出规则：首字母大写 = 公开，小写 = 私有

func DoSomething() {}   // 公开，可被外部包调用
func doInternal() {}    // 私有，仅包内可见

type MyStruct struct {    // 公开类型
    Name string         // 公开字段
    age  int            // 私有字段
}

8. 错误处理

基本模式：始终检查 error

if err != nil {
    return err    // 或 return nil, err
}

自定义错误类型

// 实现 error 接口只需要 Error() string 方法
type TypedClientError struct {
    Code    int
    Message string
    Cause   error
}

func (e *TypedClientError) Error() string { ... }
func (e *TypedClientError) Unwrap() error { return e.Cause }  // 支持 errors.Is/As

错误包装与判断

// %w 包装错误，保留原始错误链
return fmt.Errorf("acquire conversation lock failed: %w", err)

// errors.Is 判断错误类型
if errors.Is(event.Err, context.Canceled) { ... }
if errors.Is(err, context.DeadlineExceeded) { ... }

// errors.As 提取特定错误类型
var typed *TypedClientError
if errors.As(err, &typed) { ... }

项目中错误的两种处理模式

// 模式1: 直接返回 error
result.SetError(errors.New("[AgentInstance.Run] session status is running"))

// 模式2: 不返回 error，改为写入 resp.BaseResp（handler.go 常用）

defer — 延迟执行（LIFO 顺序）

    }()

// defer 常用场景
defer cancelRun()                              // 取消 context
defer ticker.Stop()                             // 停止 ticker
defer func() { if r := recover(); r != nil { ... } }()  // 捕获 panic

panic / recover

// panic: 严重错误，直接终止（类似 throw）

    panic(err.Error())
}

// recover: 必须在 defer 中调用，捕获 panic
defer func() {
    if r := recover(); r != nil {
        errMsg := fmt.Sprintf("recover from panic: %v\n stack:%s", r, debug.Stack())
        logs.CtxError(ctx, errMsg)
        result.SetError(errors.New(errMsg))
    }
}()

18. 常见模式速查

模式代码说明
错误检查 if err != nil { return err } Go 中最常见的代码行
defer 清理 defer lock.Release(ctx) 资源释放、解锁、关闭文件
命名返回值 + defer func f() (err error) { defer func() { ... }() } defer 中可修改返回值
ok 模式 if v, ok := m[key]; ok { ... } 安全地从 map 取值或类型断言
空指针检查 if req == nil || req.Field == nil { ... } 访问嵌套指针前必须检查
